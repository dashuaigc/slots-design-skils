# WILD 图标美术规范
## Slots Game WILD Symbol Art Specification

版本：1.1 | 适用范围：所有平台老虎机游戏 WILD 图标静态美术资产

---

## 规范使用说明

本文档定义了 WILD 图标美术的**行业通用标准**，涵盖视觉层级、构图、色彩、发光、文字、技术交付等维度。在以下阶段均应对照本规范：
1. 生成 AI Prompt 前（设计规划阶段）
2. 生成 AI 图像后（人工评审阶段）
3. 精修完成后（交付前终检阶段）

---

## 第零章：预览画布规范

### 0.1 评审用画布背景色

AI 生图及设计评审阶段，**禁止在纯白或纯黑背景上评审图标**。标准预览背景色：

```
评审背景色（中灰，模拟游戏卷轴典型底色）：
  标准中灰：#666666
  浅中灰：  #999999

使用原则：
  · 用 #666666 评审图标轮廓与外发光的清晰度
  · 用 #999999 评审图标暗部细节的可见性
  · 两色均测试后方可确认方案
```

> 最终交付仍为透明背景 PNG-32，此规范仅用于**设计过程评审**。

## 第一章：视觉层级规范

### 1.1 WILD 的符号层级定义

老虎机符号存在明确的视觉层级体系，WILD 必须严格符合其定位：

| 符号类型 | 视觉层级 | 视觉重量基准 |
|---------|---------|------------|
| Jackpot / Grand Prize 图标 | 最高 | 100% |
| WILD 图标（无 Jackpot 时） | 最高 | 100% |
| WILD 图标（有 Jackpot 时） | 次高 | 75-90% |
| 高价值主题图标（High Symbols） | 中高 | 55-70% |
| 低价值图标（A/K/Q/J/10） | 最低 | 30-45% |

**规范要求：**
- WILD 的综合视觉重量（发光强度 + 色彩饱和度 + 主体占比的综合评分）必须高于同画布下任何高价值图标
- 在卷轴缩略视图（约 80×80px）中，WILD 必须是第一眼被注意到的图标

### 1.2 视觉分离度

WILD 图标必须与以下符号保持清晰的视觉区分：
- **与 SCATTER 的区分**：两者不得使用相同主色调（色相差异 ≥ 30°）
- **与 BONUS 的区分**：外形语言（圆形/方形/有机形）应有明显差异
- **与高价值图标的区分**：WILD 的发光面积或强度必须明显超出所有高价值图标

## 第二章：构图规范

### 2.1 主体占比

```
画布尺寸：W × H

主体元素（含边框）安全占比：75% – 95% 的画布面积
视觉重心位置：略高于画布中心（Y 轴 42-50% 位置）
绝对不可超出：画布边界（主体应完整可见，主体超出边框的部分允许延伸至安全边距区）
```

### 2.2 主体超出边框规范（立体突破效果）

现代老虎机图标常用技巧：**主体核心部分（头部/翅膀/武器等）略微超出边框范围**，形成立体层叠效果。

```
允许超出量：边框外 3-8%（相对于画布尺寸）
超出部分必须：仍在画布安全边距内（不超出画布边缘）
超出部分建议：图标的视觉焦点元素（非底座/装饰）
图层逻辑：主体图层置于边框图层之上（Z 轴顺序）
```

### 2.3 安全边距（Safe Zone）

```
最小边距（四边）：画布尺寸的 5%
推荐边距（四边）：画布尺寸的 8-10%

512×512 画布对应：
  最小边距 = 26px（四边）
  推荐边距 = 41-51px（四边）
```

**注意：** 发光效果（Glow）可以超出安全边距，但主体轮廓线不能超出。

### 2.4 构图视角

**WILD 图标的主体应优先展示正面视角（Front-facing view）**，确保主要特征（面部、核心元素、象征物）完整面向玩家，最大化视觉冲击力与识别度。

| 视角类型 | 优先级 | 适用风格 | 注意事项 |
|---------|--------|---------|---------|
| **正视图（Front View）** | ⭐ 首选 | 所有风格 | 主体面向观者，五官/核心特征完整可见，视觉识别度最高 |
| 四分之三视角（Three-quarter） | 次选 | 写实 3D、概念原画 | 立体感强，但须确保主体面部/标志性特征仍朝向玩家 |
| 俯视图 | 不推荐 | 特殊主题 | 主体识别度明显降低，除非主题需要（如宝石俯视），否则避免使用 |

> **Prompt 描述策略：** 明确写入 `front-facing view` / `facing the viewer` / `直视观众` / `正面朝向`，确保 AI 生图遵循正面构图。

## 第三章：色彩规范

### 3.1 核心识别色数量

```
主色调：1 种（占主体面积 50% 以上）
辅助色：最多 2 种
高光/发光色：1 种（通常为主色调的亮化版本）

总计：核心色不超过 3 种
```

超过 3 种核心色会导致图标在小尺寸下视觉混乱，失去识别度。

### 3.2 色彩对比度

**主体与发光色的对比：**
- 主体暗部与外发光颜色的亮度差：≥ 60%

**与卷轴背景的对比（如已知）：**
- WILD 图标最亮区域与卷轴背景的亮度对比比：≥ 4:1
- 若卷轴背景为深色（明度 < 30%），图标应有清晰的亮色轮廓或外发光

**色彩饱和度要求：**
- WILD 的主色饱和度（HSB 中 S 值）应不低于 65%（低饱和度会降低视觉冲击力）
- 例外：刻意做旧/腐朽风格，主色饱和度可降低，但需以强烈的发光色补偿

### 3.3 禁用色彩组合

以下色彩组合会导致玩家混淆或视觉疲劳，应避免：
- ❌ WILD 与 SCATTER 使用相同主色调（色相差 < 30°）
- ❌ 图标主体纯灰色（无色彩感，难以识别）
- ❌ 主体颜色与外发光颜色使用互补色（如红色主体 + 绿色发光，圣诞节主题除外）

## 第四章：发光规范（Glow & Bloom）

### 4.1 外发光（Outer Glow）

外发光是 WILD 图标区别于普通符号的关键视觉元素，**强制要求存在**。

```
外发光参数参考（@512px 画布）：

  标准外发光（窄）：
    - 模糊半径：12-20px
    - 颜色：主色调的亮化版本（HSB 亮度 +30-50%）
    - 不透明度：70-90%
    - 混合模式：Linear Dodge (Add)

  大范围光晕（宽）：
    - 模糊半径：40-60px
    - 颜色：主色调（或偏暖/偏冷的调整版）
    - 不透明度：30-50%
    - 混合模式：Linear Dodge (Add)
```

**外发光颜色规范：**
- 不允许纯白色外发光（无主题感，无辨识度）
- 外发光颜色必须与图标主色有色相关联
- 多种主题色时，外发光取其中最亮/最饱和的一种

### 4.2 内发光（Inner Glow）

适用于有内腔、眼睛、裂缝等透光结构的图标。

```
内发光参数参考（@512px 画布）：

  内部光源核心：
    - 模糊半径：5-10px
    - 不透明度：80-100%

  内部光源漫射：
    - 模糊半径：15-25px
    - 不透明度：40-60%
```

### 4.3 发光与风格的匹配

| 风格 | 发光强度 | 备注 |
|------|---------|------|
| 写实 3D | 中（模拟物理光） | 避免卡通化过曝 |
| 2D 游戏插画 | 强（风格化） | 可以夸张 |
| 暗黑写实插画 | 弱-中（局部点光） | 大面积强光破坏氛围 |
| 像素艺术 | 弱（色块替代） | 用亮色色块模拟，不用实际模糊 |
| 东方水墨 | 极弱或无 | 靠颜色渐变与留白代替 |

## 第五章：边框规范（Frame / Border）

### 5.1 核心原则：细、简、聚焦

**边框的功能是框定区域，而非争夺视觉注意力。** 边框应细窄、简洁，让主体成为绝对焦点。

```
边框宽度：画布尺寸的 2-4%（严格限制，禁止超过 4%）

  @512px 画布：边框宽度 = 10-20px
  @1024px 画布：边框宽度 = 20-40px

边框内边距（与主体的间距）：≥ 画布尺寸的 3%
边框本身不计入主体占比，单独占用安全边距区域
```

### 5.2 推荐边框风格（按复杂度从低到高）

**所有线框类型必须为单线（Single Line）**，即一条金属线/发光线，禁止使用双线、多线并排或复合线框。

| 风格 | 特征 | 适用主题 |
|------|------|---------|
| **细金属单线框** | **单条**金属边线，简约精致 | 通用，现代风格 |
| **宝石点缀单线细框** | **单条**极细框体，四角/四边点缀小宝石 | 财富、奢华主题 |
| **发光单线框** | **单条**发光轮廓线，渐变色或单色光晕 | 科幻、魔法、能量主题 |
| **浮雕单线框** | **单条**轻浅浮雕线，厚度薄 | 历史、神话主题 |

**明确禁止：**
- ❌ 双线框或多线并排框（如内外两条线的框）
- ❌ 厚重石刻大型边框（边框视觉权重超过主体）
- ❌ 边框内外均有复杂装饰（双层复杂框）
- ❌ 边框颜色与主体颜色高度相似（失去分隔感）

### 5.3 主体突破边框（推荐效果）

主体元素（焦点部位）可微微超出边框：
- 超出量：边框外 3-8%
- 超出部分仍在画布安全区内
- 图层顺序：主体置于边框之上
- 效果：增强立体感，避免图标"被关在笼子里"的压抑感

### 5.4 无边框时的规范

- 主体轮廓必须有清晰的描边或边缘光（Rim Light）与背景分离
- 若背景透明，需确保图标在深色和浅色背景下均可识别

### 5.5 边框形状（核心更新）

**WILD 图标边框不限于矩形，可根据游戏主题与主体形态选择形状：**

| 形状 | 适用主题 | 注意事项 |
|------|---------|---------|
| **矩形** | 通用，历史/神话/现代 | 最兼容，直角可带轻微修饰 |
| **圆角矩形** | 现代奇幻、财富、卡通 | 圆角半径建议为画布宽 5-12% |
| **圆形** | 魔法、宝石、符文、东方 | 圆形框内主体须饱满填充 |
| **椭圆形** | 人物/神像垂直构图 | 长轴与主体朝向一致（通常竖向） |
| **其他多边形** | 特殊主题（盾形/钻石切面） | 须保持单线原则，避免复杂外轮廓 |

- 边框形状必须与主体形态和游戏主题协调一致
- 无论何种形状，均须遵守单线、2-4% 宽度原则（见 §5.1/5.2）
- Prompt 中明确写出形状：`rectangular frame` / `rounded rectangle border` / `circular frame` / `oval border`

### 5.6 线框内背景规范（核心更新）

**边框内区域（主体所在内容区）的背景处理原则：**

**核心要求：**
- 背景必须与主体形成清晰的明暗或色彩对比，让主体突出醒目
- 背景需与游戏主题及氛围协调，**禁止使用与主题无关的背景**
- **禁止花哨、复杂的背景**：背景视觉复杂度必须远低于主体

**推荐处理方式：**

| 背景类型 | 描述 | 适用主题 |
|---------|------|---------|
| 主题径向暗渐变 | 深色调为主的径向渐变，中心略亮，与主体色相呼应 | 暗黑/神话/恐怖 |
| 材质底面 | 石材/金属/皮革等与主题匹配的微纹理底面，低对比细节 | 历史/写实/奇幻 |
| 光雾氛围 | 薄雾/光效渐变，隐约可见主题氛围，不喧宾夺主 | 魔法/奇幻/科幻 |
| 深色单色 | 深色纯色，无纹理，突出主体的最简方案 | 任何主题均可作备选 |

**明确禁止：**
- ❌ 鲜艳/高饱和度复杂背景（喧宾夺主，与主体竞争视觉焦点）
- ❌ 白色或高明度浅色底（主体暗部轮廓融入背景，失去立体感）
- ❌ 与主体主色高度接近的背景（失去前景/背景对比）
- ❌ 与主题毫无关联的背景（如古代神像主题使用科幻光栅背景）

**Prompt 描述策略：**
- 英：`dark [theme-appropriate] atmospheric background inside frame, subtle [texture/gradient], strong value contrast with main subject, no busy patterns`
- 中：`边框内[主题元素]暗色氛围背景，微渐变底色，与主体形成强烈明暗对比，禁止花哨纹样`

## 第六章："WILD"文字规范

### 6.1 位置规范（核心更新）

**WILD 文字应置于图标底部区域，并可略微超出边框下边缘。**

```
WILD 文字垂直位置：图标画布高度的 70-90%（即位于下部 1/4 区域）
文字向下超出边框量：文字高度的 10-30%（向下超出边框下缘，不超出画布安全边距）
文字水平宽度：略超出边框内宽，两端各超出边框约 2-5%（形成横向突破感）
文字水平位置：水平居中
```

**位置示意：**
```
┌─────────────────────┐  ← 画布上边缘
│                     │
│      [ 主体 ]       │  ← 主体核心（可微超边框上缘）
│                     │
│   ┌─────────────┐   │  ← 边框上边缘
│   │             │   │
│   │  [ 主体 ]   │   │  ← 主体主要部分
│   │             │   │
│  [    WILD    ] │   │  ← 文字两端轻微超出边框左右边缘（约2-5%）
│   └─────────────┘   │  ← 边框下边缘（WILD文字向下骑跨约15-25%）
└─────────────────────┘  ← 画布下边缘
```

> **注意：** 文字横向超框是一种轻微突破，视觉上应感觉文字"略大于框"，而非明显溢出。文字底部下沉幅度（纵向超出）通常比横向超出更明显。

### 6.2 文字尺寸

```
"WILD" 文字高度上限：图标画布高度的 20%
"WILD" 文字高度下限：图标画布高度的 10%

@512px 画布：
  推荐文字高度：65-85px
  最小文字高度：52px（低于此值在游戏中难以阅读）

@1024px 画布：
  推荐文字高度：130-170px
  最小文字高度：100px
```

### 6.3 字体风格规范（核心更新）

**WILD 字体必须与游戏主题风格匹配，禁止使用无装饰通用字体（Arial、Helvetica 等）。**

| 主题风格 | 推荐字体方向 | 字形特征 |
|---------|------------|---------|
| 神话/历史/古典 | 石刻铭文体 | 厚重衬线，雕刻质感，棱角分明 |
| 暗黑/恐怖/哥特 | 哥特装饰字 | 尖锐细腿，哥特花饰，锋利感 |
| 魔法/奇幻/科幻 | 符文/能量字 | 发光轮廓，流动笔触，解构感 |
| 财富/喜庆/节日 | 圆润粗体/装饰金字 | 饱满圆润，金色立体，喜庆感 |
| 休闲/卡通/冒险 | 手写/卡通粗体 | 活泼不规则，有笔画趣味 |
| 东方/水墨 | 毛笔楷书/篆体 | 书法气息，笔锋明显 |

**Prompt 中描述字体的策略：**
- 明确提及字体风格（如：gothic letterform / runic engraving / bold serif with ornate decoration）
- 描述字体的材质处理（如：gold bas-relief / carved stone / glowing energy outline）
- 说明字体与主体的关系（如：floating above / carved into surface / seamlessly integrated）

### 6.4 文字立体感与颜色规范（核心更新）

**WILD 文字必须具有明显的三维立体感，禁止平面字效。**

**立体感要求：**

```
必须体现的3D效果元素（至少包含其中2项）：
  · 厚度面（Side faces）：字体有可见的侧面厚度
  · 高光面（Highlight surface）：顶面受主光源照射形成亮面
  · 阴影面（Shadow face）：底面或侧面形成阴影
  · 浮雕感（Bas-relief）：文字从背景表面凸起

禁止：
  ❌ 2D 平面文字（无厚度感）
  ❌ 仅有投影（Drop shadow）但无立体厚度
  ❌ 描边字（Stroke only，无立体体积）
```

**颜色规范：**

```
主色调：金色系（首选）
  · 标准黄金：#D4A017 ～ #FFD700（暖金/明金）
  · 古铜金：  #8B6914 ～ #C8A84B（古朴/氧化金）
  · 玫瑰金：  #B76E79 ～ #E8A087（华丽/时尚风）

允许替代色（主题驱动）：
  · 白银/镀铬：当主题需要冷色金属质感时
  · 血红/暗金：当主题为恐怖/诅咒时（需结合金色高光）
  · 魔法光色：当主题为纯粹魔法/能量时（蓝白/紫白能量字）

禁止：
  ❌ 纯灰色文字（无贵重感）
  ❌ 纯白色文字（缺乏主题感）
  ❌ 纯黑色文字（低辨识度）
```

**Prompt 描述策略：**
- 英文：`3D embossed gold lettering` / `thick golden relief text with visible side faces` / `high-relief cast gold inscription`
- 中文：`3D立体浮雕金色铭文` / `厚重黄金铸造文字，有明显厚度侧面` / `高浮雕金色铭文，主光面亮金，侧面古铜暗金`

### 6.5 文字可读性

**最小显示尺寸测试（64×64px）：**
- 在 64×64px 缩略图中，"WILD" 文字必须可辨认出至少 3 个字母
- 如不可辨认，需增加文字描边宽度或简化文字装饰

**描边规范：**
```
描边宽度：字体大小的 4-8%
描边颜色：与文字颜色高对比（通常为深色 + 亮色文字，或亮色 + 深色文字）

@80px 字体高度：
  最小描边宽度：3-4px
  推荐描边宽度：5-7px
```

**字距（Letter Spacing）：**
- 推荐：字体大小的 5-15%
- 不允许字母重叠（Tracking 不为负值）

## 第九章：剪影清晰度规范

### 9.1 剪影测试方法

**剪影测试**：将图标填充为纯黑色（无任何内部细节），在白色背景上查看轮廓。如果此时仍能判断出这是什么主题元素，则剪影合格。

```
合格标准：
  ✅ 纯黑剪影下仍能识别主体形态（角色/物体/符号）
  ✅ 剪影有明显的方向感与重量感
  ✅ 剪影与游戏其他符号的剪影有可区分的形态差异

不合格：
  ❌ 剪影过于方正/圆润，与其他符号高度相似
  ❌ 剪影内部空洞太多，导致形态破碎
  ❌ 剪影边缘过于复杂，在小尺寸下变成噪点
```

### 9.2 剪影在各尺寸下的表现要求

| 尺寸 | 剪影要求 |
|------|---------|
| 512px | 全部细节轮廓可辨，可识别具体形态 |
| 128px | 主体大轮廓清晰，次要轮廓可见 |
| 64px | 主体形态可判断，细节允许消失 |
| 32px | 外发光 + 整体剪影形状可辨即达标 |

### 9.3 不同构图策略的剪影特征

| 构图策略 | 剪影特征 | 设计建议 |
|---------|---------|---------|
| 单一主体居中 | 饱满圆润，边缘简洁 | 避免过多凸起细节破坏轮廓 |
| 垂直构图 | 上窄下宽或上宽下窄，有明显方向感 | 确保最窄处不低于画布宽度 15% |
| 放射构图 | 从中心向外扩散，锯齿感强 | 光芒数量建议 4-8 条，避免过密 |
| 对称有机形 | 左右对称，自然流畅 | 对称轴偏差不超过 5px（@512px）|

## 第十章：视觉复杂度规范

### 10.1 复杂度层级定义

老虎机图标的视觉复杂度分为三个层级，设计时需与游戏平台和目标用户匹配：

| 层级 | 特征 | 适用场景 |
|------|------|---------|
| **简洁（Simple）** | 色块 ≤ 5 种，细节适度，剪影优先 | 移动端优先、休闲游戏、低分辨率设备 |
| **均衡（Balanced）** | 色块 5-10 种，有纹理但不繁琐 | 主流 5×3 游戏，兼顾移动与桌面 |
| **精细（Detailed）** | 色块 10+ 种，丰富纹理与细节 | 桌面端为主、高端游戏、展示用途 |

### 10.2 各平台复杂度建议

| 游戏平台 | 建议层级 | 最小显示尺寸参考 |
|---------|---------|--------------|
| 移动端竖屏（320-375px宽） | 简洁 | 约 50-60px 图标 |
| 移动端横屏（568px+宽） | 均衡 | 约 70-90px 图标 |
| 平板端 | 均衡-精细 | 约 100-140px 图标 |
| 桌面端 | 精细 | 约 120-200px 图标 |

### 10.3 AI 生图的复杂度控制

在 Prompt 中控制复杂度的描述策略：

**降低复杂度（简洁风格）：**
- 使用：`clean design`, `bold shapes`, `minimal details`, `strong silhouette`, `flat areas of color`
- 避免：`highly detailed`, `intricate patterns`, `complex texture`, `busy background`

**提升复杂度（精细风格）：**
- 使用：`highly detailed`, `intricate engravings`, `complex texture`, `rich surface details`, `8K resolution`
- 辅助：`subsurface scattering`, `micro detail`, `fine filigree`, `ornate decoration`
