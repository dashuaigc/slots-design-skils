# 输出归档格式规范（核心）
# 供 SKILL.md 第 8 步调用。定义存档路径、命名规则、各平台文件格式、设计简报归档。
# 修订流程见 output-format-revision.md（仅修订模式加载）。

---

## 存档路径

```
Prompt 文件：../../outputs/prompts/
设计简报：  ../../outputs/briefs/
```

自检循环完成后**自动存档**，无需用户确认。

---

## 核心原则：纯 Prompt 输出

- **每个平台单独一个文件**。
- 文件内**只包含 Prompt 内容**，不含需求摘要、自检报告、设计说明等。
- 用户可直接复制粘贴到对应平台使用。

---

## 各平台输出格式

### Midjourney — 仅英文 `.md`

```markdown
[英文 Prompt 内容，含 --ar --v 等参数]
```

示例文件名：`midjourney_caishen_toon3d_20260303_001.md`

---

### Nano Banana — 结构化 JSON 片段 `.md`

按用户选择语言：仅英文 → 一段英文JSON；仅中文 → 一段中文JSON；两种都要 → 两段。

```md
// English Version
```json
{
  "subject": "[角色：视角、姿势、表情、特征]",
  "style": "[渲染风格：写实等级、细节密度、边缘锐度]",
  "material": "[主材质：粗糙度/金属度倾向]",
  "lighting": "[光照：方向/类型/色温]",
  "color": "[配色：主色调、饱和度要求、外发光颜色]",
  "composition": "[构图：主体占比、视觉重心、超边框量]",
  "frame": "[边框：样式、形状、宽度比例]",
  "background": "[框内背景：氛围、明暗对比要求]",
  "wild_text": "[WILD文字：字体、主题融合、3D效果、位置超框量]",
  "negative": "[排除的元素和效果]",
  "specs": "[技术规格：画布比例、背景色]"

```
```

示例文件名：`nb2_caishen_toon3d_20260303_001.md`

## 文件命名规则

```
格式：[平台缩写]_[主题关键词]_[风格代码]_[YYYYMMDD]_[NNN].[ext]
```

| 平台 | 缩写 |
|------|------|
| Midjourney | `midjourney` |
| Nano Banana | `nb2` |
| 即梦 | `jimeng` |
| Stable Diffusion | `sd` |
| Flux | `flux` |
| DALL-E | `dalle` |

**风格代码：** `real3d` / `toon3d` / `illustration` / `flat2d` / `gothic` / `concept` / `ink` / `anime` / `cyberpunk` / `pixel`

**序号自动递增：**
1. 读取 `outputs/prompts/` 下所有文件名
2. 提取同前缀文件数量
3. 新文件序号 = 现有数量 + 1，补齐为 3 位数
4. 目录为空或无同前缀文件 → 序号 `001`

## 设计简报归档

### 触发时机

Q31 用户确认"无需修改"后，**在生成 Prompt 之前**立即写入简报文件。

### 简报命名

```
outputs/briefs/[主题关键词]_[风格代码]_[YYYYMMDD]_[NNN]_brief.md
```
（序号与 Prompt 文件保持一致）

### 简报内容结构

```markdown
# WILD 图标设计简报 — [主题] · [日期]

## 游戏基础
- 主题：[Q1]  目标玩家：[Q2]  目标市场：[Q3]  卷轴布局：[Q4]

## WILD 规格
- 符号格比例：[Q5]  画布比例：[Q6]  画布背景色：[Q7]  符号层级：[Q8]

## 主体设定
- 主体类别：[Q9]  具体身份：[Q9b]
- 服装装扮：[Q9c]  体态动作：[Q9d]  面部气质：[Q9e]  标志道具：[Q9f]
- 构图形态：[Q10]  灵感来源：[Q11]

## 美术方向
- 视觉复杂度：[Q12]  氛围基调：[Q14]  绘画风格：[Q15]
- 主材质：[Q16]  光照风格：[Q17]  特定元素：[Q18]

## 色彩与情绪
- 主色调：[Q19]  情绪基调：[Q20]

## WILD 文字
- 字体风格：[Q21]  文字位置：[Q22]  衬线风格：[Q23]  关联装饰：[Q24]

## 边框与背景
- 边框设计：[Q25]  边框形状：[Q26]  框内背景：[Q27]  视觉复杂度确认：[Q28]

## 参照与限制
- 竞品参照：[Q29]  内容限制：[Q30]

## 选定平台与模型
[每行一个平台及其选定模型]

## 竞品分析摘要
[联网搜索 3-5 条关键结论]

## Q31 补充设定
[所有补充内容逐条列出；无补充则写"无"]
```

**补充整合规则：**
- 每条 Q31 补充必须同步更新简报对应字段
- 原始补充记录写入「Q31 补充设定」章节
- 最终简报体现所有补充后的完整方案

## 收尾询问

存档完成后告知用户每个文件路径，然后弹窗：

```
AskUserQuestion：
  question: "设计方案已完成并存档，你希望接下来做什么？"
  multiSelect: true
  options:
    - 修改/补充当前 Prompt 内容（重新生成）
    - 生成不同视角/光照/材质的变体 Prompt
    - 为配套符号（SCATTER/高价值图标）生成风格一致 Prompt
    - 拿到生图结果后进行二次图像自检
```

选"修改/补充当前 Prompt 内容"时，读取 `references/output-format-revision.md` 进入修订流程。
