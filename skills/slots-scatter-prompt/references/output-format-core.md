# 输出格式规范（SCATTER）

供 SKILL.md 第 9 步调用。定义 Prompt 文件归档和设计简报的格式规范。

---

## 核心原则

- **纯 Prompt 输出**：每个平台单独一个文件，只包含 Prompt 内容，不含需求摘要
- **存档路径**：Prompt → `../../../../outputs/prompts/`，简报 → `../../../../outputs/briefs/`

---

## 各平台 Prompt 文件格式

| 平台 | 格式 | 语言 | 说明 |
|------|------|------|------|
| **Midjourney** | `.md` | 仅英文 | 含 `--ar --v` 等参数 |
| **Nano Banana** | `.md` | 中英文各一版 | 结构化JSON片段，分段输出 |
| **即梦** | `.md` | 仅中文 | 不超过 800 字 |
| **其他** | `.md` | 按平台偏好 | SD/Flux/DALL-E等 |

---

## 文件命名规则

```
格式：[平台缩写]_[主题关键词]_[风格代码]_[YYYYMMDD]_[NNN].[ext]

平台缩写：mj（Midjourney）/ nb2（Nano Banana）/ jm（即梦）
主题关键词：来自游戏主题，小写字母+下划线，最长20字符
风格代码：real3d / toon3d / illustration / flat2d / gothic / concept / ink / anime / pixel
序号：读取现有文件数，新序号 = 现有数量 + 1，补齐3位数

示例：
  mj_caishen_scatter_toon3d_20260306_001.md
  nb2_caishen_scatter_toon3d_20260306_001.md
  jm_caishen_scatter_toon3d_20260306_001.md
```

> 注：SCATTER 文件名中建议加入 `scatter` 关键词以区分 WILD 文件。
