# 修订流程规范
# 供 SKILL.md 第 10 步（修订模式）调用。
# 正常归档流程见 output-format-core.md。

---

## 触发条件

以下任一情况触发修订模式：
- 用户在收尾询问中选择"修改/补充当前 Prompt 内容"
- Prompt 输出后，用户说明效果不满意或需要修改

---

## Step R1 — 定位并加载简报文件

1. 扫描 `../../../../outputs/briefs/` 目录。
2. 当前会话只有一个 SCATTER 简报文件 → 自动加载，不弹窗。
3. 存在多个 SCATTER 简报文件 → 弹窗让用户选择：
   ```
   question: "发现多个 SCATTER 设计简报，请选择要基于哪个版本进行调整："
   options: [列出所有 scatter_*_brief.md 文件名，最多 3 项 + Other]
   ```

---

## Step R2 — 展示当前完整方案

读取简报文件，以结构化文本直接输出（不弹窗）：

```
📋 当前 SCATTER 设计方案（来自：[文件名]）
──────────────────────────
[简报全文内容]
──────────────────────────
```

---

## Step R2.5 — 确认操作类型

展示方案后立即弹窗：

```
AskUserQuestion：
  question: "你希望对当前方案做什么调整？"
  options:
    - 补充内容（在现有方案基础上增加新信息/元素）
    - 修改内容（替换现有方案中的某项设定）
    - 两者都有（既补充又修改）
```

- 选**补充内容** → Step R3a
- 选**修改内容** → Step R3b
- 选**两者都有** → 先 R3a，完成后 R3b

---

## Step R3a — 补充内容收集（自由输入）

```
AskUserQuestion：
  question: "请描述你想补充的内容（例如：增加折射效果、添加特定装饰、补充发光细节等）"
  options:
    - 补充主体细节/材质/折射/特效
    - 补充特殊视觉效果（光效/粒子/环境元素等）
    - 补充限制条件（避免某些元素、特定平台合规要求等）
```

记录用户"其他"中的描述，整合到对应简报字段。

---

## Step R3b — 修改意向收集（针对 Batch）

```
AskUserQuestion：
  question: "你希望修改哪个方向的设定？"
  options:
    - 主体/构图/风格/材质（对应 Batch 3.1）
    - 色彩与情绪（对应 Batch 3.2）
    - SCATTER 标签文字（对应 Batch 3.4，若有文字）
    - 边框与背景（对应 Batch 3.5）
```

用户选"其他"时，按自由描述定位对应 Batch。

---

## Step R4 — 针对性重问对应 Batch

仅针对 R3b 选中的方向，复用 `questionnaire-p3.md` 中对应 Batch 重新问询（其余字段保持不变）。

完成后弹窗确认：
```
question: "还有其他需要调整的方向吗？"
options:
  - 没有了，确认全部调整，进入汇总确认
  - 还需要修改其他方向（返回 Step R3b）
  - 还需要补充其他内容（返回 Step R3a）
```

---

## Step R5 — 汇总确认并写入新简报

**1. 输出完整更新方案**，修改项用 `→` 标注：

```
📋 更新后的 SCATTER 设计方案
──────────────────────────
[完整简报内容，修改项标注 → 变更]
──────────────────────────
变更项：
  - [字段名]：[旧值] → [新值]
  - [补充项]：[新增内容]
```

**2. 弹窗最终确认：**

```
AskUserQuestion：
  question: "以上是更新后的完整方案，确认无误后将存档并重新生成 Prompt"
  options:
    - 确认无误，存档并重新生成 Prompt（Recommended）
    - 还需要调整（返回 Step R2.5）
```

**3. 用户确认后写入新版简报文件：**

```
路径：../../../../outputs/briefs/scatter_[主题]_[风格]_[YYYYMMDD]_[NNN]_brief.md
序号：与即将生成的新 Prompt 文件序号相同
内容：完整简报，末尾附「修订记录」章节（说明从哪版修订、改动了哪些字段）
```

**旧 brief 文件保留不删除。**

---

## Step R6 — 重新生成 Prompt 并存档

基于新简报，执行主流程步骤 7-8：

1. 生成差异化 Prompt（按 `prompt-spec.md`）
2. 自检循环（对照 `scatter-art-core.md` 12条约束）
3. 存档 Prompt 文件（新序号，与 brief 一致）

完成后告知用户全部文件路径：

```
✅ 修订完成

新版简报：outputs/briefs/scatter_[新文件名]_brief.md
新版 Prompt：
  - outputs/prompts/[平台]_[主题]_scatter_[风格]_[日期]_[NNN].md
  - ...

旧版文件（保留）：
  - outputs/briefs/scatter_[旧文件名]_brief.md
  - outputs/prompts/[旧文件名].md
```
