# 弹窗问询执行协议

供 SKILL.md 第 2、6 步调用。定义弹窗工具使用规则与降级策略。

---

## 工具选择

- 优先使用 `AskUserQuestion` 工具执行弹窗问询。
- 工具不可用时按"降级策略"执行。

## 核心规则

| 规则 | 说明 |
|------|------|
| 单题提交 | 每次调用 `questions` 数组长度 **必须为 1** |
| 选项数量 | 每题提供 2-3 个互斥选项；客户端自动追加 `Other` 作为"其他" |
| 推荐标注 | 推荐项放第一位，标签末尾加 `(Recommended)` |
| 多选字段 | 对业务多选（如目标平台），拆分为连续单选子题 |
| 条件追问 | 条件子字段必须追问。例："是否要边框=是"后 → 继续问边框样式、材质、粗细 |
| WILD 文字组合追问 | 必须依次问：文字结构、材质、主题融合元素、超框比例、特效处理 |
| "其他"处理 | 用户选"其他"时，记录原文后正常进入下一题 |
| 顺序锁定 | 第一阶段和第三阶段都必须逐题确认，不允许更改提问顺序 |
| 进度提示 | 每批回答后简短告知："已收到 [当前批次/总批次]" |

## 降级策略（弹窗工具不可用时）

- 改为普通文本逐题提问，保持**一题一问**。
- 每题显式给出 `1-3` 主选项 + `4. 其他（填写...）`，提示可回复"其他 + 内容"。
- 不因降级而跳过任何问题。

## 阶段与批次对照

| 阶段 | 批次 | 内容简述 |
|------|------|---------|
| 第一阶段 | Batch 1.1 | 游戏基础信息（主题/玩家/市场/卷轴） |
| 第一阶段 | Batch 1.2 | WILD 基础规格（符号格/画布/背景色/层级） |
| 第一阶段 | Batch 1.3 | 主体与灵感设定（类别/形态/灵感） |
| 第一阶段 | Batch 1.4 | 视觉复杂度与平台选择 |
| 第三阶段 | Batch 3.1 | 创意方向确认（氛围/风格/材质/光照/特定元素） |
| 第三阶段 | Batch 3.3 | 色彩与基调（主色调/情绪） |
| 第三阶段 | Batch 3.4 | WILD 文字设计（字体/位置/衬线/装饰） |
| 第三阶段 | Batch 3.5 | 边框与视觉细节（边框/形状/框内背景/复杂度） |
| 第三阶段 | Batch 3.6 | 竞品与限制（竞品参照/内容限制） |
| 第三阶段 | Batch 3.7 | 模型最终选择（每选中平台一题） |

**注意：** 第三阶段选项内容不是固定的，由 `search-strategy.md` 智能分析生成。
