# 输出归档格式规范

供 SKILL.md 第 10 步调用。定义存档路径、命名规则、各平台文件内容格式。

---

## 存档路径

```
../../outputs/prompts/
```

自检循环完成后**自动存档**，无需用户确认。

---

## 核心原则：纯 Prompt 输出

- **每个平台单独一个文件**。
- 文件内**只包含 Prompt 内容**，不含需求摘要、自检报告、设计说明等额外信息。
- 用户拿到文件后可直接复制粘贴到对应平台使用。

---

## 各平台输出格式

### Midjourney — 仅英文 `.md`

文件内容**只有**英文 Prompt，可直接粘贴到 Discord/Midjourney。

```markdown
[英文 Prompt 内容，含 --ar --v 等参数]
```

**示例文件名：** `midjourney_caishen_toon3d_20260303_001.md`

---

### Nano Banana — 结构化 JSON 片段 `.md`

文件为 Markdown，内含结构化 JSON 代码块。按用户选择语言输出对应片段：
- 仅英文 → 一段英文 JSON 片段
- 仅中文 → 一段中文 JSON 片段
- 中英文都要 → 两段独立 JSON 片段，分别用注释注明语言

每段 JSON 将 Prompt 按维度拆分为独立字段（字段名统一用英文），便于按维度修改复用：

```md
// English Version
```json
{
  "subject": "[角色描述：视角、姿势、表情、身材特征]",
  "style": "[渲染风格：写实等级、细节密度、边缘锐度]",
  "material": "[主材质：皮肤/金属/布料，粗糙度/金属度倾向]",
  "lighting": "[光照：光源数量/方向/类型/色温]",
  "color": "[配色：主色调、饱和度要求、外发光颜色]",
  "composition": "[构图：主体占比、视觉重心、超边框量]",
  "frame": "[边框：样式、形状、宽度比例]",
  "background": "[框内背景：处理方式、氛围、明暗对比要求]",
  "wild_text": "[WILD文字：字体风格、主题融合、3D效果、位置超框量]",
  "negative": "[明确排除的元素和效果]",
  "specs": "[技术规格：画布比例、背景色]"
}
```

// 中文版本
```json
{
  "subject": "...",
  "style": "...",
  "material": "...",
  "lighting": "...",
  "color": "...",
  "composition": "...",
  "frame": "...",
  "background": "...",
  "wild_text": "...",
  "negative": "...",
  "specs": "..."
}
```
```

**示例文件名：** `nb2_caishen_toon3d_20260303_001.md`（后缀 `.md`）

---

### 即梦 — 仅中文 `.md`

文件内容**只有**中文 Prompt（不超过 800 字），可直接粘贴到即梦平台。

```markdown
[中文 Prompt 内容]
```

**示例文件名：** `jimeng_caishen_toon3d_20260303_001.md`

---

### 其他平台 — 按平台偏好 `.md`

文件内容为该平台最优输入语言的 Prompt。

**示例文件名：** `[平台缩写]_[主题]_[风格]_[日期]_[序号].md`

---

## 文件命名规则

```
格式：[平台缩写]_[主题关键词]_[风格代码]_[YYYYMMDD]_[NNN].[ext]
```

### 平台缩写

| 平台 | 缩写 |
|------|------|
| Midjourney | `midjourney` |
| Nano Banana | `nb2` |
| 即梦 | `jimeng` |
| Stable Diffusion | `sd` |
| Flux | `flux` |
| DALL-E | `dalle` |

### 主题关键词

从游戏主题提取，小写英文，空格用 `_` 替代，最长 20 字符。
示例：`caishen` / `tumbbad` / `olympus` / `dragon`

### 风格代码

| 代码 | 含义 |
|------|------|
| `real3d` | 写实 3D 渲染 |
| `toon3d` | 风格化/卡通 3D |
| `illustration` | 手绘厚涂插画 |
| `flat2d` | 2D 扁平插画 |
| `gothic` | 哥特暗黑水彩 |
| `concept` | 概念原画风格 |
| `ink` | 水墨/东方风格 |
| `anime` | 日系动漫插画 |
| `cyberpunk` | 赛博朋克/霓虹科幻 |
| `pixel` | 像素/复古像素艺术 |

### 日期与序号

- 日期：当天 `YYYYMMDD`
- NNN：三位数序号，从 `001` 起自动递增

### 自动递增逻辑

```
1. 读取 outputs/prompts/ 目录下所有文件名
2. 提取同前缀（[平台]_[主题]_[风格]_[日期]）的文件数量
3. 新文件序号 = 现有数量 + 1，补齐为 3 位数
4. 如目录为空或无同前缀文件，序号 = 001
```

### 命名示例

```
midjourney_caishen_toon3d_20260303_001.md
nb2_caishen_toon3d_20260303_001.md
jimeng_caishen_toon3d_20260303_001.md
```

## 会话汇总文档归档

### 触发时机

第四阶段用户确认"无需修改"后，**在生成 Prompt 之前**立即执行。

### 目录结构

```
outputs/
├── prompts/                                          # 各平台 Prompt 文件
└── briefs/                                           # 设计汇总文档
    └── [主题]_[风格]_[YYYYMMDD]_[NNN]_brief.md
```

文件命名规则与 Prompt 文件前缀保持一致，末尾加 `_brief` 后缀以区分：`[主题关键词]_[风格代码]_[YYYYMMDD]_[NNN]_brief.md`（同一批次序号相同）。

### design-brief.md 内容结构

```markdown
# WILD 图标设计简报 — [主题] · [日期]

## 游戏基础
- 主题：[Q1答案]
- 目标玩家：[Q2]  目标市场：[Q3]
- 卷轴布局：[Q4]

## WILD 规格
- 符号格比例：[Q5]  画布比例：[Q6]
- 画布背景色：[Q7]  符号层级：[Q8]

## 主体设定
- 主体类别：[Q9]  构图形态：[Q10]
- 灵感来源：[Q11]

## 美术方向
- 视觉复杂度：[Q12]
- 氛围基调：[Q14]  绘画风格：[Q15]
- 主材质：[Q16]  光照风格：[Q17]
- 特定元素：[Q18]

## 色彩与情绪
- 主色调：[Q19]  情绪基调：[Q20]

## 汇总确认弹窗

存档前需执行一次汇总确认：

```
AskUserQuestion：
  question: "以上设计需求汇总已生成，请确认是否有需要补充或修改的信息？"
  multiSelect: true
  options:
    - 确认无误，继续生成 Prompt
    - 需要修改[某项]（请在"其他"中说明）
    - 需要补充[某项]（请在"其他"中说明）
```

如用户需修改或补充，允许**单次补正**后再次确认。

---

## 收尾询问

存档完成后告知用户每个文件的完整路径，然后弹窗：

```
AskUserQuestion：
  question: "设计方案已完成并已存档，你希望接下来做什么？"
  multiSelect: true
  options:
    - 修改/补充当前 Prompt 内容（重新生成）
    - 生成不同视角/光照/材质的变体 Prompt
    - 为配套符号（SCATTER/高价值图标）生成风格一致 Prompt
    - 拿到生图结果后进行二次图像自检
```

选"修改/补充当前 Prompt 内容"时，进入下方**修订流程**。

---

## 修订流程

### 触发条件

以下任一情况触发：
- 用户在收尾询问中选择"修改/补充当前 Prompt 内容（重新生成）"
- Prompt 文件输出后，用户通过消息说明效果不满意、需要修改或补充

---

### Step R1 — 定位并加载简报文件

1. 扫描 `../../outputs/briefs/` 目录。
2. 当前会话只有一个简报文件 → 自动加载，不弹窗。
3. 存在多个简报文件 → 弹窗让用户选择：
   ```
   question: "发现多个设计简报，请选择要基于哪个版本进行调整："
   options: [列出所有简报文件名，最多 3 项 + Other]
   ```

---

### Step R2 — 展示当前完整方案

读取简报文件，以结构化文本直接输出给用户（不弹窗），格式：

```
📋 当前设计方案（来自：[文件名]）
──────────────────────────
[简报全文内容]
──────────────────────────
```

---

### Step R2.5 — 确认操作类型（必须优先问）

**展示方案后，立即弹窗询问本次操作类型：**

```
AskUserQuestion：
  question: "你希望对当前方案做什么调整？"
  options:
    - 补充内容（在现有方案基础上增加新信息/元素）
    - 修改内容（替换现有方案中的某项设定）
    - 两者都有（既补充又修改）
```

- 选**补充内容** → 进入 Step R3a（自由补充输入）
- 选**修改内容** → 进入 Step R3b（针对性 Batch 重问）
- 选**两者都有** → 先走 Step R3a，完成后走 Step R3b

---

### Step R3a — 补充内容收集（自由输入）

弹窗让用户描述要新增的内容：

```
AskUserQuestion：
  question: "请描述你想在现有方案上补充的内容（例如：增加火焰特效、添加特定道具、补充姿势细节等）"
  options:
    - 补充主体细节/姿势/表情（请在"其他"中描述）
    - 补充特殊视觉效果（光效/粒子/环境元素等）
    - 补充限制条件（避免某些元素、特定平台合规要求等）
```

记录用户在"其他"中的自由描述，整合到对应的简报字段中。

---

### Step R3b — 修改意向收集（针对 Batch）

弹窗询问要修改哪个方向（单选，可多轮）：

```
AskUserQuestion：
  question: "你希望修改哪个方向的设定？"
  options:
    - 主体/构图/风格/材质（对应 Batch 3.1）
    - 色彩与情绪（对应 Batch 3.3）
    - WILD 文字设计（对应 Batch 3.4）
    - 边框与背景（对应 Batch 3.5）
```

用户选"其他"时，记录自由描述，按描述定位对应 Batch。

---

### Step R4 — 针对性重问对应 Batch

仅针对 Step R3b 选中的方向，**复用 questionnaire.md 中对应 Batch 的题目**重新弹窗问询（其余字段保持不变）。

完成后弹窗确认：
```
question: "还有其他需要调整的方向吗？"
options:
  - 没有了，确认全部调整，进入汇总确认
  - 还需要修改其他方向（返回 Step R3b）
  - 还需要补充其他内容（返回 Step R3a）
```

---

### Step R5 — 汇总确认并输出更新后的简报文件

所有补充/修改收集完毕后：

**1. 输出完整更新方案汇总**，以结构化文本展示变更前后对比：

```
📋 更新后的设计方案
──────────────────────────
[展示完整简报内容，修改项用 → 标注变更]
──────────────────────────
变更项：
  - [字段名]：[旧值] → [新值]
  - [补充项]：[新增内容]
```

**2. 弹窗最终确认：**

```
AskUserQuestion：
  question: "以上是更新后的完整方案，确认无误后将存档并重新生成 Prompt"
  options:
    - 确认无误，存档并重新生成 Prompt（Recommended）
    - 还需要调整（返回 Step R2.5）
```

**3. 用户确认后，写入新版简报文件**（序号与 Prompt 文件保持一致）：

```
输出路径：../../outputs/briefs/[主题]_[风格]_[YYYYMMDD]_[NNN]_brief.md
序号规则：与即将生成的新 Prompt 文件序号相同（如 Prompt 为 002，则 brief 也为 002）
内容：完整简报，末尾附「修订记录」章节，说明本次从哪个版本修订、改动了哪些字段
```

**旧 brief 文件保留不删除。**

---

### Step R6 — 重新生成 Prompt 并存档

基于新 brief 文件，从主流程步骤 8 重新执行：

1. 生成差异化 Prompt（步骤 8）
2. 自检循环（步骤 9）
3. 存档 Prompt 文件（步骤 10），新文件序号与 brief 一致

完成后告知用户全部文件路径：

```
✅ 修订完成

新版简报：outputs/briefs/[新文件名]_brief.md
新版 Prompt：
  - outputs/prompts/[平台]_[主题]_[风格]_[日期]_[NNN].md
  - ...

旧版文件（保留）：
  - outputs/briefs/[旧文件名]_brief.md
  - outputs/prompts/[旧文件名].md / .json
```
